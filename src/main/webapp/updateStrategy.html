Minimal cleanup - Only undo controlled file modifications. All other files will be kepts as they are. This can have unwanted side effects but might be a desired if build artefacts are expected to be kept inside workspace.

Standard cleanup - Undo changed files and remove any files that are not under version control. Ignored files will be preserved. Recommended.

Full cleanup - Undo changed files and remove all files that are not present in the repository. Private and ignored files will be deleted resulting in a pristine workspace state.

Delete workspace - The workspace will be wiped out with all its content. A new repository will be created. All files will be redownloaded ensuring workspace is in pristine state. resulting in a pristine workspace state. This operation can take very long time depending on the size of the repository. Not recommended.
